# Dashboard連動 学習スコープ（固定）

## 目的
- このプロジェクトでは、**Master画面（S1〜S10）のCRUDは実装済み**とする。
- 現在の学習・実装対象は、**Dashboard画面との連動**に限定する。

## In Scope（実装してよい）
- Dashboardで使うデータをController経由でprops化して渡す。
- Dashboard側の固定配列・ダミーデータを、Master由来の実データ参照へ置き換える。
- 連動確認に必要な最小限の表示・フィルタ調整。

## Out of Scope（実装しない）
- Master側CRUDの新規実装・仕様変更。
- S1〜S10の既存受け入れ済み挙動の変更。
- 旧デモ（`legacy-demo/`）の削除や構成変更。

## 運用ルール
- 作業開始時に対象を明示する（例: `対象セクション: S1`）。
- 1ブランチ1目的を守る。ブランチ名は `codex/dashboard-sX-<topic>` を推奨。
- スコープ外の発見はコード変更せず、影響メモだけ残す。

## 新チャット開始テンプレ（コピペ用）
```md
対象セクション: Sx

前提:
- Master画面（S1〜S10）のCRUDは実装済み
- 今回の対象はDashboard連動のみ
- Master側は原則変更しない

進め方:
- 5分タスクで1手ずつ進める
- 私が実装し、あなたは「次の1手」とレビューを担当する
- スコープ外変更は実装せず影響メモに残す
```
