# S5影響メモ（請求稼働の月表示方針）

## 対象
- セクション5（請求稼働実績）CRUD実装時点での、月表示方針の整理。

## 判定
- S5は `billing_work_logs.work_month` の全件を基準に月列を表示する。
- 月列順は昇順（古い月→新しい月）を維持する。
- 初期表示および案件切替時は最新月列が見える位置へ自動スクロールする。
- S5の入力列は「1か月 = 稼働(h) / 単価(円) の2子列」構成にする。
- キーボード操作は `Enter=下移動`、`Tab=右移動`、矢印キー移動を維持する。
- 未保存状態はセル単位で可視化し、保存/破棄/再読込時にハイライトをクリアする。
- 保存payloadは従来どおり `entries[{ user_id, work_month, billed_hours, billing_rate }]` を維持する。

## 補足
- S6〜S10は既存の `monthMaster`（固定7か月）を継続利用する。
- 全セクション共通の月マスタ統一は別タスクとする。
