# S7影響メモ（メンバー原価の表示切替）

## 対象
- セクション7（メンバー原価）をS5/S6同等操作（暦年/決算年度12か月表示、キーボード操作、未保存可視化）へ拡張。

## 判定
- S7の月列はフロントで「選択した暦年/決算年度の12か月」を生成する。
- 保存先は `staff_monthly_results` とし、`user_id + work_month` 単位で一括保存する。
- 入力刻みは以下に固定する。
  - 給与: 1000円単位
  - 法定福利費: 1000円単位
  - 福利厚生費: 10円単位
  - 賞与: 1000円単位

## S6連動時の注意点
- S7の社内原価(h)は `s6BusinessDaysForMonth` を参照して計算する。
- S6に営業日数が保存されていない月は既存仕様どおり `1` 日として計算される。
- そのため、S7で表示中の月にS6未保存月が含まれる場合、社内原価(h)は暫定値になる。

## S8への影響（調査のみ）
- S8のUI/保存仕様は今回変更しない。
- S8は従来の `visibleMonths`（固定7か月）前提を維持し、S7ロジックと分離する。

## 今回の実装スコープ外
- S8保存API化。
- S7/S8共通の月マスタ統一。
- S9/S10への12か月表示展開。
