# S4影響メモ（会計CSV月範囲）

## 対象
- セクション4（会計データ / freee CSVインポート）実装時点での、他セクション月表示範囲との差分整理。

## 判定
- S4は `monthly_accounting_data.work_month` を基準に月表示する。
- S5〜S10は既存の `monthMaster`（固定7か月）を継続利用する。

## 今回の実装スコープ外
- S5〜S10の月表示範囲をS4に合わせて拡張する対応。
- 全セクション共通の月マスタ統一。

## 次ブランチ候補（影響調査のみ）
- S5以降のセクションでもDB起点の月定義へ寄せる場合、UI列数・検証ロジック・既存データ移行手順を合わせて定義する。
