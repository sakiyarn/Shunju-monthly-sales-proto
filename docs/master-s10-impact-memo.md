# S10影響メモ（請求調整CRUD移行）

## 対象
- セクション10（請求調整）をローカル保存実装から `billing_adjustments` のCRUDへ移行。
- UIを「固定高一覧 + 暦年フィルタ + 行単位編集モーダル」に変更。

## 判定
- `original_month` は必須で保持し、監査・メモ自動補助入力に利用する。
- 金額は円整数運用（負値許容）とし、controllerで整数正規化する。
- 新規選択肢は「有効メンバー + 進行中案件」のみ許可する。
- 既存行で無効メンバー/終了案件参照がある場合は表示・更新を許可し、参照維持を優先する。
- 保存は `bulk_sync`（create/update/delete 同時反映）で transaction 一括処理とする。

## UI/操作方針
- 一覧は読み取り専用とし、編集は行単位モーダル経由のみ許可する。
- 一覧領域は固定高（内部スクロール）で、件数増加時もカード高さを拡張しない。
- 編集途中の離脱時は未保存警告を維持する（S9のガードと共存）。
- 金額表示は `¥` + 3桁区切り（表示欄のみ）とする。

## S9への影響確認
- S9 `s9Modal.dirty` を `hasUnsavedChanges` から外さない。
- S9の一覧・モーダル・検索・年度フィルタ機能は変更しない。

## Dashboard連携（今回スコープ外）
- Dashboard本体のコメント描画ロジックは改修しない。
- 既存互換のため、S10保存成功時のみ localStorage ミラーを書き込み続ける。
