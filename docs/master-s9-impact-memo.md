# S9影響メモ（特定経費CRUD移行）

## 対象
- セクション9（特定経費）をダミー配列実装から `directed_expenses` / `directed_expense_assignments` のCRUDへ移行。
- UIを「固定高一覧 + 暦年フィルタ + 検索 + 行単位編集モーダル」に変更。

## 判定
- S9の `project_ids` は数値ID（`number[]`）で統一する。
- フロント項目名は `name` ではなくDBカラムに合わせて `description` を使用する。
- 金額は円整数運用（小数不可）とし、controllerで整数正規化する。
- 更新時は「active案件のみ差し替え + 既存closed案件紐づけ保持」を適用する。
- 終了案件は新規選択不可、既存紐づけは表示のみ保持する。

## UI/操作方針
- 一覧領域は固定高（内部スクロール）で、件数増加時もカード高さを拡張しない。
- 暦年フィルタと検索で過去データの参照性を確保する。
- 編集は行単位のモーダル経由のみ許可し、直接入力による誤操作を防止する。
- 編集途中のモーダル離脱や画面遷移時は未保存警告を維持する。

## S10への影響（調査のみ）
- S10の型/保存方式は今回変更しない。
- S9のみを `number[]` ベースに移行し、S10の既存ローカル保存ロジックは維持する。

## 将来課題（今回スコープ外）
- `MasterController#index` propsの増加に伴い、Inertia lazy propsへの段階移行を検討する。
- S6〜S10の月定義統一（暦年/決算年度含む）は別タスクで扱う。
